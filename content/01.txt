Mitchell Hashimoto’s New Way of Writing Code
從自學程式到創辦 HashiCorp：一段改變雲端基礎設施的旅程

Mitchell Hashimoto 是 HashiCorp 的共同創辦人，也是 Terraform、Vagrant、Vault 等知名開源工具的創造者。他的工具改變了整個雲端基礎設施管理的方式。在這次訪談中，他回顧了自己的程式生涯、HashiCorp 的誕生，以及開源、雲端與 AI 對軟體工程未來的影響。

本文依照影片內容順序整理，完整梳理 Mitchell 的經歷與觀點。

1. 一切的開始：一個 12 歲自學寫程式的少年

Mitchell 的程式旅程開始得非常早。

他在 12、13 歲左右開始自學寫程式，最初的動機和許多程式設計師一樣——遊戲。

但很快他發現自己更喜歡 Web 開發。當時網路仍是非常早期的年代，Google 甚至還沒有出現。他開始學習：

PHP

Perl

基本網站開發

由於年紀太小、沒有錢買書，他只能透過 網路上的開源程式碼學習。

有一段故事非常經典：

他把 PHP manual 的前幾章列印出來（30–40頁），每天走路上學時反覆閱讀。
某一天，他突然理解了 PHP 中的 $ 符號代表 變數（variables）。

這個瞬間成為他的突破點。

理解變數之後，他的學習速度突然加快，開始能真正寫程式。

2. 青少年時期：接案、模仿網站、賺第一筆錢

在高中時期，Mitchell 已經每天寫程式。

他做過很多事情：

建立遊戲相關網站

建立論壇

嘗試複製網站功能

模仿 PayPal 的付款系統

甚至還：

假裝自己 18 歲在接案網站接工作。

他接過的案子包括：

圖片上傳功能

小型網站開發

每個案子可能只有 $50 或 $100，但這讓他開始用程式賺錢。

3. 程式設計是一條孤獨的路

在當時，喜歡電腦並不是一件酷的事情。

Mitchell 說：

當時如果你喜歡電腦，幾乎等於社交死亡。

因此：

他在學校 完全不提自己寫程式

這幾乎是他的秘密

但他在網路上找到了同好：

MSN Messenger

AOL Messenger

各種論壇

這些線上朋友後來很多都成為他長期的朋友。

4. 第一份真正的工作：Ruby on Rails

在大學（University of Washington）期間，他收到一封改變人生的 email。

有人問他：

「你想成為 Ruby on Rails 工程師嗎？」

當時他只會 PHP，完全不會 Ruby。

但他還是接受了這份工作。

這家公司是一間 Rails 顧問公司，專門幫客戶建立 MVP。

他在那裡做過：

YouTube 類型網站

電商網站

慈善平台

這段經歷讓他快速接觸到各種不同的技術與產品。

5. 一個奇怪的導師改變了他的方向

在這家公司，有一位非常神秘的工程師。

特徵：

長髮

不用滑鼠

使用 Linux

坐在角落

Mitchell 對這個人非常好奇。

某天這位工程師做了一件事情：

他拔掉 Mitchell 的滑鼠。

然後說：

「你以後都不能用滑鼠了。」

Mitchell 花了一週時間學會 完全用鍵盤操作電腦。

之後他又學會：

terminal

screen

SSH

Linux package manager

這讓他開始進入 基礎設施 (infrastructure) 的世界。

6. 一個失敗的大學研究專案

在大學時期，他加入了一個研究計畫：

Seattle Project

目標是建立一個類似：

Folding@Home

分散式運算網路

讓世界各地的電腦可以貢獻算力。

Mitchell 的任務是：

建立節點管理與基礎設施系統

但他 完全失敗了。

然而這個失敗非常重要。

他在筆記本中寫下：

自己無法完成的原因：

無法 declaratively 管理資源

無法建立私有網路

無法管理節點

這些問題後來變成：

HashiCorp 的產品藍圖。

7. 一封 Email 開啟創業

他的研究主管 Arman 看了這本筆記。

幾週後發了一封 email：

「要不要一起創業？」

Mitchell 兩分鐘內回覆：

「好。」

HashiCorp 就這樣誕生了。

8. 第一個成功產品：Vagrant

在顧問公司工作時，他遇到一個問題：

每個專案都有不同的開發環境。

建立環境可能需要 半天時間。

因此他做了一個工具：

Vagrant

目標是：

「雙擊就能啟動一個開發環境。」

技術：

VirtualBox

虛擬機

這解決了開發環境不可重現的問題。

9. 雲端時代的到來

2010 年左右，AWS 剛推出不久。

當時：

EC2 非常不穩定

服務很少

幾乎只有 S3 可靠

但 Mitchell 感覺：

這就是未來。

並不是因為市場分析，而是：

它感覺是更好的方式。

10. HashiCorp 的產品版圖

HashiCorp 逐步推出一系列工具。

1️⃣ Vagrant

開發環境管理

2️⃣ Packer

建立機器映像 (machine images)

3️⃣ Consul

服務發現 (service discovery)

4️⃣ Terraform

Infrastructure as Code

描述基礎設施：

這段文字 = 整個 AWS infrastructure

執行後：

建立 VPC

Subnets

Instances

Gateways

甚至可以一鍵刪除。

5️⃣ Vault

Secrets 管理

例如：

密碼

API key

信用卡資料

PII

6️⃣ Nomad

工作排程系統

在多台機器上自動部署應用。

11. 開源與商業模式的困境

HashiCorp 一開始 完全沒有商業模式。

前四年：

幾乎沒有收入

只有開源專案

投資人仍然支持，因為：

GitHub stars

下載量

開發者採用

顯示出 產品市場契合度 (product-market fit)。

12. 第一個商業模式失敗

他們第一個商業產品叫：

Atlas

但失敗原因是：

必須同時使用所有 HashiCorp 工具。

企業不願意這樣做。

此外還有另一個問題：

企業內部不知道 誰要付錢。

例如：

security 部門

networking 部門

infrastructure 部門

大家互相推責任。

13. 轉折點：Open Core 模型

某次董事會後，他們意識到方向錯了。

於是決定：

全部重來。

新的模式：

Open Core

基礎版本開源

企業功能閉源

並且：

每個產品獨立商業化

第一個成功產品是：

Vault Enterprise

企業需要：

secrets replication

高可用

支援服務

這讓公司開始有收入。

14. Terraform 為何爆紅

很多人認為 Terraform 成功是因為：

「第一個進市場」

Mitchell 說：

Terraform 其實是第七個。

真正原因是：

他到處演講

不斷參加 conference

與開發者直接交流

甚至 9 年內每 8 天就旅行一次。

他在飛機上寫程式：

下載 GitHub issues

分解成 15 分鐘任務

飛機上逐一解決

降落後一次 push 30 個 commit。

15. VMware 差點買下 HashiCorp

創業兩年後：

VMware 想收購公司。

最初報價：

$20M

Mitchell 和 Arman 只有 23 歲。

這是巨額金錢。

但他們想到：

賣掉公司後，夢想可能會死。

最後他們提出：

$100M

VMware 董事會投票：

差一票未通過。

如果那票通過：

Terraform 可能永遠不會出現。

16. HashiCorp 上市

HashiCorp 在 2021 年 IPO。

上市準備：

需要一年以上

模擬財報電話會議

法律高度保密

甚至 Mitchell 的父母都不知道他要上市。

17. 他對雲端巨頭的看法

Mitchell 說：

AWS 當時非常傲慢。

有一種感覺：

「我們隨時可以做一個產品毀掉你的公司。」

但同時 AWS 也有：

世界上最優秀的工程師之一。

18. AI 對開源的衝擊

Mitchell 提出一個重要觀點：

AI 可能改變開源模式。

因為 AI 可以：

自動產生 pull request

產生看起來合理但錯誤的程式碼

而開源原本依賴：

信任。

未來可能變成：

Default deny → 必須先建立信任。

結語

Mitchell Hashimoto 的故事展示了一個典型但極具影響力的技術創業旅程：

自學程式

發現基礎設施問題

開源工具

找到商業模式

改變整個產業

HashiCorp 的成功不是因為第一個進市場，而是：

長期專注解決真實問題。

正如 Mitchell 所說：

好的工程師不是只寫程式，而是理解系統與限制。

而這種思維，正是現代軟體工程的核心。
